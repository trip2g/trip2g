# Do Next

## 1. Проверь состояние репозитория

```bash
git status
git diff --stat
git log --oneline -5
```

- Посмотри что недавно менялось
- Если много незакоммиченных изменений (>5 файлов или >100 строк) → предложи сделать коммит сейчас

## 2. Прочитай `docs/current_tasks.md`

## 3. Если есть задача [IN PROGRESS]:

- Прочитай связанную документацию (указана в контексте)
- Найди текущий шаг (первый незавершённый `[ ]`)
- Прочитай заметки
- Продолжи выполнение

## 4. Если нет активных задач:

- Спроси пользователя что делаем
- Прочитай связанную документацию
- Создай задачу по формату:

```markdown
## [IN PROGRESS] Название

### Контекст
Что делаем и зачем (2-3 предложения).
Связанные доки: docs/xxx.md

### План
- [ ] Прочитать связанную документацию
- [ ] Шаг 1
- [ ] Шаг 2
- [ ] ...
- [ ] Запустить make test && make lint, исправить ошибки

### Заметки
(пусто пока)
```

**ВАЖНО:** Последний шаг плана всегда — `make test && make lint`.

## 5. Начни работу над первым незавершённым шагом

## 6. После завершения задачи

Запусти:
```bash
make test && make lint
```

Если есть ошибки:
- Ошибки в твоём коде → исправь
- Ошибки в других местах → предложи пользователю исправить сейчас
