# Channel Export Tool

Exports Telegram channel messages to Markdown format for use in Obsidian vaults.

## Setup

### 1. Get Telegram API Credentials

1. Go to https://my.telegram.org/apps
2. Login with your phone number
3. Create a new application
4. Copy your `api_id` and `api_hash`

### 2. Get OpenAI API Key

1. Go to https://platform.openai.com/api-keys
2. Create an API key

### 3. Set Environment Variables

```bash
export TELEGRAM_API_ID="your_api_id"
export TELEGRAM_API_HASH="your_api_hash"
export OPENAI_API_KEY="your_openai_api_key"
```

## Usage

```bash
go run ./cmd/channelexport \
  --channel-id=-1001234567890 \
  --output-path=/path/to/obsidian/vault \
  --limit=20
```

### Parameters

- `--channel-id` (required): Numeric Telegram channel ID (including the minus sign for channels)
- `--output-path` (required): Directory where markdown files will be saved
- `--limit` (optional): Maximum number of messages to export (default: 20)

### Finding Channel ID

You can find the channel ID using bots like @username_to_id_bot or by inspecting the channel link.

## Authentication

On first run, the tool will prompt for:
1. **Phone number**: Enter in international format (e.g., `+1234567890`)
2. **Verification code**: Enter the code sent to your Telegram app

The session is saved to `session.json` for subsequent runs. Once authenticated, you won't need to login again unless you delete the session file.

## Output Format

Each message is exported as a separate markdown file with:

### Filename

- Title generated by OpenAI GPT-5-nano (3-5 words, key idea of post)
- Uses simple text generation with system/user message pattern
- Spaces are preserved in filenames: `"ĐśĐľĐą Ń„Đ°ĐąĐ».md"`
- Duplicate titles get message ID appended: `"ĐśĐľĐą Ń„Đ°ĐąĐ» - 123.md"`

### Tracking File (`posts.json`)

The tool maintains a `posts.json` file in the output directory to track processed messages:

```json
{
  "channel_id": "-1001234567890",
  "posts": [
    {
      "message_id": 123,
      "filename": "AI Generated Title.md"
    },
    {
      "message_id": 124,
      "filename": "Another Post Title.md"
    }
  ]
}
```

**Incremental Sync**: On subsequent runs, the tool reads `posts.json` and skips messages that have already been downloaded. This allows you to:
- Re-run the tool to fetch new messages
- Increase `--limit` to fetch more history without re-downloading existing posts
- Safely interrupt and resume downloads

## Limitations (Current Version)

- Text-only export (media files not downloaded yet)
- Interactive authentication only (no bot token support)
- Channel must be in first 100 dialogs
